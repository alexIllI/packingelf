# PackingElf 測試說明

這份文件是給開發者、測試者與部署人員的檢查清單。  
目標是確認 `Client`、`Host`、installer 與同步流程都能正常運作。

## 測試前準備

- 一台安裝 `Host` 的電腦
- 一台或多台安裝 `Client` 的電腦
- 或在同一台 dev 電腦同時測 `Host + Client`
- 有效的 `myacg` 帳號
- 可用的印表機或虛擬 PDF 印表機

## 安裝測試

### Client 安裝

1. 執行 installer
2. 只勾選 `包貨小精靈 Client`
3. 完成安裝
4. 確認可以正常開啟 app

### Host 安裝

1. 執行 installer
2. 只勾選 `包貨小精靈 Host`
3. 完成安裝
4. 確認可以正常開啟 Host app

### 同機安裝

1. 執行 installer
2. 同時勾選 `Client` 與 `Host`
3. 確認兩個 app 都能正常開啟

## Client 功能測試

### 初次啟動

確認：

- 設定檔會自動建立
- 本機 SQLite 會自動建立
- 記錄檔資料夾會自動建立

### myacg 帳號設定

1. 開啟 `設定 -> myacg 帳號`
2. 新增一筆帳號
3. 測試登入
4. 將模式切換成 `自動登入`
5. 重新開啟 app
6. 確認 app 會依設定登入

### 列印頁流程

目前正式流程：

1. 先輸入發票號碼
2. 按 `Enter`
3. 焦點跳到貨單尾碼
4. 輸入 5 碼尾碼
5. 按 `Enter`
6. 開始抓單與列印

成功後應確認：

- 列印成功
- 訂單寫入本機資料庫
- 買家名稱有抓到
- 總金額有抓到
- 表格中可看到最近 24 小時內的訂單

### 例外情境

請至少測一次：

- 查無訂單
- 訂單已取消
- 門市關轉
- 已取貨
- 重複訂單重新列印

## Pending Table 測試

1. 在待處理區輸入貨單尾碼
2. 可選填備註
3. 點 `新增待處理`
4. 確認資料寫進本機資料庫
5. 確認 pending 不會同步到 Host

## Host 功能測試

### Host 啟動

確認：

- Host 視窗可正常開啟
- 可看到主機網址、配對碼、資料庫路徑
- `總覽 / 貨單 / Client / 匯出` tab 可正常切換

### Client 連線

1. 先開 Host
2. 再開 Client
3. 在 Client 設定主機網址與配對碼
4. 確認 Host 的 `Client` 頁可看到已連線 client

### 同步測試

1. 在 Client 成功列印一筆訂單
2. 確認 Host 貨單列表有收到資料
3. 檢查欄位：
   - 貨單號碼
   - 發票號碼
   - 買家
   - 日期
   - 總金額
   - 狀態

## 匯出測試

在 Host 的 `匯出` 頁確認：

- 單日匯出正常
- 日期區間匯出會產生每天一份檔案
- 月份匯出正常
- 可開啟匯出資料夾

## 記錄檔與問題排查

### Client

確認首頁可以：

- 開啟目前記錄檔
- 開啟記錄資料夾
- 開啟資料庫位置

### Host

確認 Host 端也能從介面找到：

- 資料庫位置
- 匯出資料夾

## Release 前最少檢查

正式發版前，至少完成這些測試：

- installer 可正常安裝 Client
- installer 可正常安裝 Host
- Client 可登入 `myacg`
- 正常訂單可列印並寫入本機 DB
- 正常訂單可同步到 Host
- Host 可匯出 Excel
- 升級安裝不會清掉既有使用者資料
